# 软著代码文档生成工具需求说明书 (V2.0)

## 1. 项目概述
本工具旨在自动化扫描指定目录下的多种编程语言源代码。通过**非破坏性读取（只读）**方式，在内存中完成代码清洗（去注释、去空行）并实时排版，最终导出为符合标准的 Word 文档。

---

## 2. 核心功能需求

### 2.1 多目录源代码扫描
* **只读扫描**：程序仅对指定的源代码目录进行 `Read` 操作，严禁任何 `Write` 或 `Modify` 操作。
* **多路径支持**：支持同时指定一个或多个源代码根目录。
* **后缀过滤**：支持自定义后缀白名单（如 `.go`, `.cpp`, `.java`, `.dart` 等）。

### 2.2 健壮的编码兼容性（防乱码）
* **自动检测**：读取前利用 `charset_converter` 配合 BOM 检测、UTF-8 验证、GBK 统计检测等算法自动识别文件编码（UTF-8/GBK/ANSI等）。
* **内存转码**：文件内容加载至内存后统一转为 Unicode 字符串，确保 Word 导出时中文不乱码，不影响物理文件本身编码。

### 2.3 内存流清洗逻辑 (不改变源文件)
在数据流向 Word 缓冲区时，实时执行以下过滤动作：
* **正则去注释**：利用正则表达式匹配并过滤掉单行/多行注释，不保存至临时文件。
* **行过滤**：在内存中剔除所有 `strip()` 后为空的行。
* **预处理预览**：清洗后的代码仅存在于程序变量中，直至写入 Word。

### 2.4 Word 文档排版规范
* **行数控制**：
    * **默认设置**：每页固定 **50 行** 代码（支持设置）。
    * **实现手段**：通过设置固定行高或计算分页符实现。
* **页眉配置**：
    * **居中内容**：显示“软件名称 + 版本号”（信息由界面输入）。
    * **右侧内容**：显示动态页码。
* **样式设置**：
    * 字体：采用等宽字体（推荐 Consolas 或 Courier New）。
    * 布局：标准 A4 纸张，页眉与正文间距适中。

---

## 3. 处理流程逻辑图



1. **输入阶段**：获取用户选择的目录及软件元数据（名称、版本）。
2. **读取阶段**：以 `rb` (二进制只读) 模式读取文件 -> 检测编码 -> 解码为文本。
3. **清洗阶段 (Memory Only)**：
    - `text = re.sub(comments_regex, '', text)`
    - `lines = [line for line in text.splitlines() if line.strip()]`
4. **组装阶段**：将清洗后的行数据按每 50 行逻辑切分，写入 Word 的 Section。
5. **输出阶段**：生成并保存 `.docx` 文档。

---

## 4. 安全与性能保证
* **物理隔离**：程序逻辑确保不调用任何文件写入 API 指向源码目录。
* **异常处理**：若遇到无法读取的特殊格式文件，跳过并记录日志，不中断任务。
